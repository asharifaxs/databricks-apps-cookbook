# Databricks Apps Cookbook — High-Level Overview

> **Note:** This document was generated by Cursor (Composer 2.5) on 2026-06-11. See [`README.md`](./README.md) for provenance and disclaimers.

## Purpose

This repository is an **official-style reference cookbook** for building **data and AI applications on [Databricks Apps](https://docs.databricks.com/en/dev-tools/databricks-apps/index.html)**. It pairs:

1. **Runnable demo apps** (Streamlit, Dash, Reflex, FastAPI)
2. **Documentation site** ([apps-cookbook.dev](https://apps-cookbook.dev/)) with copy-paste recipes

Each “recipe” shows how to connect an app to Databricks platform services — tables, volumes, ML serving, workflows, and more — with permissions, dependencies, and working code.

---

## Intended Audience

| Audience | Why this repo fits |
|----------|-------------------|
| **App developers** (primary) | Python web devs building UIs or APIs on Databricks |
| **Data engineers / analytics engineers** | People who know SQL/Delta but need app patterns |
| **Data scientists / ML engineers** | Model serving, vector search, GenAI, multimodal LLM examples |
| **Platform / solutions engineers** | Auth (OBO vs service principal), secrets, external connections, deployment |

It assumes comfort with Python and basic Databricks concepts (Unity Catalog, SQL warehouses). It is **not** a Databricks 101 or a full platform operations guide.

---

## Problem It Solves

Building on Databricks Apps means wiring together many services (Unity Catalog, SQL warehouses, model endpoints, jobs, secrets, Lakebase, etc.) with correct **auth**, **permissions**, and **SDK usage**. That integration knowledge is scattered across docs.

This cookbook **centralizes proven patterns** as:

- Interactive “Try it” demos
- Copy-paste code snippets
- Explicit requirements (permissions, resources, `requirements.txt`)

You can deploy a full demo app to your workspace or run locally with the Databricks CLI — then lift individual recipes into your own app.

## Key Concepts Demonstrated

1. **Databricks Apps runtime** — deploy Python apps with managed compute, service principal auth, and `app.yaml` entrypoints
2. **Unity Catalog integration** — read/edit Delta tables, list catalogs/schemas, volumes upload/download
3. **SQL connectivity** — Databricks SQL Connector + SDK `WorkspaceClient`
4. **Lakebase (OLTP)** — PostgreSQL-compatible transactional DB with OAuth token refresh
5. **Model serving & GenAI** — LLM, traditional ML, multimodal, vector search
6. **AI/BI** — embed Lakeview dashboards, Genie conversational analytics
7. **Workflows** — trigger jobs and retrieve task outputs
8. **Compute** — connect to shared clusters and serverless
9. **Authentication patterns** — current user headers, on-behalf-of-user (OBO) vs service principal
10. **Governed external access** — Unity Catalog connections, secrets scopes, MCP servers
11. **Local vs deployed dev** — `DATABRICKS_HOST`, CLI OAuth, behavior differences when not on Apps

## Recommended Learning Path (New to Databricks Apps)

```
Phase 1: Understand the platform
  └─ Read docs/docs/intro.md
  └─ Read Databricks Apps docs (linked from intro)
  └─ Read docs/docs/deploy.md

Phase 2: Run locally (fast feedback loop)
  └─ Clone repo → cd streamlit/
  └─ venv + pip install -r requirements.txt
  └─ databricks auth login + export DATABRICKS_HOST
  └─ streamlit run app.py
  └─ Try: Introduction → Read a Databricks table → Get current user

Phase 3: Core data patterns
  └─ Tables: read → edit → Lakebase (lakebase_read.py)
  └─ Volumes: upload → download
  └─ Unity Catalog: list catalogs/schemas

Phase 4: AI & automation
  └─ Invoke model → Vector search → Genie → Embed dashboard
  └─ Trigger job → Retrieve job results

Phase 5: Production concerns
  └─ OBO vs service principal
  └─ Secrets + external connections + MCP
  └─ Connect cluster / serverless

Phase 6: Choose your framework
  └─ Compare same recipe across streamlit/, dash/, reflex/
  └─ For APIs: fastapi/ (Lakebase + pagination patterns)

Phase 7: Deploy to workspace
  └─ Git folder in workspace → Compute → Apps → Deploy streamlit/
  └─ Re-test recipes that need Apps-only features (user headers, some connections)
```

## One-Paragraph Summary

The **Databricks Apps Cookbook** is a hands-on reference library for building Python data and AI applications on Databricks Apps. It ships four framework implementations (Streamlit, Dash, Reflex, FastAPI) of the same integration “recipes” — reading and writing Unity Catalog tables and volumes, calling model serving and vector search, embedding Genie and dashboards, triggering workflows, and handling auth, secrets, and external connections — each with interactive demos, copy-paste code, and documented permissions. A Docusaurus site at apps-cookbook.dev publishes the documentation, while the repo itself can be deployed as a live cookbook app in your Databricks workspace or run locally with the Databricks CLI.
